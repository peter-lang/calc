# calc вЂ” documentation

`calc` is a command-line calculator written in Rust. Beyond ordinary
arithmetic it understands **exact rational arithmetic**, **physical units**
(length, area, volume, mass, temperature, time) with conversions, and **live
currency conversion** using exchange rates from the Hungarian National Bank
(MNB).

It runs as an interactive REPL (no arguments) or evaluates a single
expression passed on the command line.

```console
$ cargo run
>> 1/2 + 1/3
0.833333вЂ¦
>> 5 m 10 cm to cm
510 cm
>> 95 f to c
35 C
>> 100 EUR to USD
108.42вЂ¦  USD

$ cargo run -- 2^10
1024
```

## How it fits together

Input flows through a classic interpreter pipeline:

```
text в”Ђв”Ђв–¶ Lexer в”Ђв”Ђв–¶ [Token] в”Ђв”Ђв–¶ Parser в”Ђв”Ђв–¶ Node (AST) в”Ђв”Ђв–¶ eval() в”Ђв”Ђв–¶ Value в”Ђв”Ђв–¶ Display
        (regex)              (packrat)                  (op enums)
```

A `Value` is a [`Number`](numbers.md) paired with an optional
[`Unit`](units.md). Numbers are kept exact (`Int` / `Rational`) for as long as
possible and only fall back to `Float` when an operation forces it.

## Building and running

```bash
cargo build            # debug build
cargo run              # REPL
cargo run -- 2 + 2     # one-shot evaluation
cargo test             # unit tests (rational.rs, unit.rs)
```

Cached data (REPL history, currency rates) lives in the platform cache
directory resolved by [`files::cache`](../src/files.rs); see
[currency.md](currency.md).

## Installing

Via the Homebrew tap (macOS / Linux):

```bash
brew install peter-lang/tap/calc
```

Or with the shell installer attached to each release:

```bash
curl -LsSf https://github.com/peter-lang/calc/releases/latest/download/calc-installer.sh | sh
```

## Releasing

Releases are driven by [`cargo-dist`](https://opensource.axo.dev/cargo-dist/):
pushing a `vX.Y.Z` tag triggers `.github/workflows/release.yml`, which builds
binaries for macOS (arm64/x86_64) and Linux (arm64/x86_64), publishes a GitHub
Release (tarballs, checksums, shell installer), and pushes the Homebrew formula
to `peter-lang/homebrew-tap`.

The manual steps before tagging:

```bash
# 1. bump version in Cargo.toml, then refresh the lockfile
cargo build

# 2. regenerate the bundled third-party license file (deps may have changed)
cargo about generate about.hbs -o THIRD-PARTY-LICENSES.html

# 3. commit the version bump + regenerated licenses, then tag
git commit -am "chore: release vX.Y.Z"
git tag vX.Y.Z
git push && git push origin vX.Y.Z
```

`calc` is licensed Apache-2.0; `THIRD-PARTY-LICENSES.html` (generated by
[`cargo-about`](https://github.com/EmbarkStudios/cargo-about), config in
`about.toml`) is bundled into every release archive to satisfy dependency
attribution. Regenerating it is kept manual on purpose, since releases are
infrequent.
